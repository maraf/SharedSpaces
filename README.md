# SharedSpaces

A pure API service for managing shared spaces, allowing clients to collaborate via QR-code-based invitations and JWT-authenticated access.

---

## Design Decisions

| Topic | Decision |
|---|---|
| IDs | All IDs are **GUIDs** |
| PIN lifecycle | Once a JWT token is created from a PIN, the PIN is **dropped** — used PINs are not stored |
| Token format | **JWT** with decodable claims: `display_name`, `server_url`, `space_id` |
| QR code format | URL string encoded as `server_url:space_id:pin` |
| Display name | Client picks a display name **before joining** a space; it cannot be changed per-space; the UI supports defining a primary/current display name |
| Access type | **MVP: Read/Write for all members**; future versions may introduce configurable access levels |
| Server project | **Pure API** — no server-rendered UI |
| Space item GUIDs | **Generated by the client**, not the server |

---

## Architecture

### Tech Stack

- **ASP.NET Core** — pure Web API (no Razor Pages / MVC views)
- **Entity Framework Core** — data persistence
- **JWT** (`System.IdentityModel.Tokens.Jwt`) — authentication tokens
- **QR code** — generated client-side from the `server_url:space_id:pin` string

## API Endpoints

| Method | Route | Description | Auth |
|---|---|---|---|
| `POST` | `/spaces` | Create a new shared space | None |
| `GET` | `/spaces/{spaceId}` | Get space info | JWT |
| `POST` | `/spaces/{spaceId}/pins` | Generate a PIN (returned with QR string `server_url:space_id:pin`) | None |
| `POST` | `/spaces/{spaceId}/tokens` | Exchange PIN + display name for a JWT; PIN is deleted | None |
| `GET` | `/spaces/{spaceId}/items` | List items in the space | JWT |
| `PUT` | `/spaces/{spaceId}/items/{itemId}` | Upsert an item (client-provided GUID) | JWT |
| `DELETE` | `/spaces/{spaceId}/items/{itemId}` | Delete an item | JWT |

---

## JWT Token Claims

The token is a standard JWT that the client can decode without a server round-trip:

```json
{
  "sub": "<member_id>",
  "display_name": "Alice",
  "server_url": "https://myserver.example.com",
  "space_id": "<space_guid>",
  "exp": 1234567890
}
```

---

## Join Flow

1. Space owner calls `POST /spaces/{spaceId}/pins` → server generates a short numeric PIN, stores it (hashed, with TTL), and returns the QR string `server_url:space_id:pin`.
2. Joining client scans the QR code → extracts `server_url`, `space_id`, and `pin`.
3. Client chooses a **display name** (shown in the UI as the primary/current display name).
4. Client calls `POST /spaces/{spaceId}/tokens` with `{ pin, display_name }` → server validates PIN, creates a `SpaceMember`, issues a JWT, and **deletes the PIN**.
5. Client stores the JWT and uses it as a Bearer token for all subsequent requests.
